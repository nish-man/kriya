Lever 3 · live multi-agent system

Which brokers are worth the team's time?

Shared governance layer

Built to stay honest inside a regulated bank.

Both levers touch either existing banking customers or intermediary relationships, so the governance is strict by design. The same guardrail and eval discipline applies across both, keeping a human accountable for every outbound action.

Guardrails
Specific to a regulated lender's growth function. Not generic AI safety.
GuardrailTriggerAction
No auto-contact
human-in-loop
Any lever produces a customer or broker to approach
Output is a ranked list only. A human RM decides whether and how to make contact. The agent never messages anyone.
Credit / growth separation
deterministic
A credit signal is used in cross-sell scoring
Credit signals rank growth priority only. They never feed a lending decision, kept structurally separate from the credit-assessment agents.
Rank, never permanently gate
deterministic
A broker or account scores low this cycle
Low score sets priority order only. Nothing is removed from the pool permanently; every entity is re-scored each cycle.
Risk pre-check gate
llm check
An account shows obvious credit red flags
Held back from the RM list entirely and shown in a separate "held back" panel with the reason, so no unsuitable account is ever surfaced for outreach.
Transparent reason codes
llm check
Any ranking is produced
Every entry carries a plain-language reason so a human can see why it ranked as it did and overrule with full context.
Model degradation alert
llm eval

Trigger a prompt review before the next run. Log the failed cases. Never update prompts without re-running the full suite.
